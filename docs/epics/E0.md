# E0 - Foundation (M0)

> Brief for the foundation epic: repo shape, the two storage/config interfaces,
> and the tooling contract every later session inherits. PRD §5-§7, §12 are the
> source of truth; this restates what later epics consume.

## Goal

Stand up a runnable, forkable skeleton: `uv` project, `ruff`+`mypy`+`pytest`
gate, the `Repository` and `ConfigProvider` interfaces with their v1 impls, a
stubbed CLI with run manifests, and cloneability hygiene. After E0 the repo is
green on `make check` and every later epic fills stubs behind stable contracts.

## Repo shape

```
grepify/
  models.py              # domain records = PRD §6 column set (backend-neutral)
  clock.py               # injectable clock (determinism, PRD §5 / S8)
  paths.py               # data-dir layout (JSONL truth + cache location)
  errors.py              # GrepifyError / ConfigError / RepositoryError
  run.py                 # run_id + run-manifest I/O (F-OPS-04)
  cli.py                 # typer app: ingest|extract|trends|digest|build|validate|health|backfill
  repository/            # Repository interface + JSONL+SQLite impl + schema DDL + commit helper
  config/                # ConfigProvider interface + filesystem-YAML impl + pydantic schemas
sources/                 # groups/*.yml, keywords.yml, settings.yml   (config, committed)
data/                    # JSONL truth (committed) + grepify.db cache (gitignored)
tests/                   # pytest + fixtures/config/
docs/                    # prd.md, architecture.md, setup.md, epics/, templates
```

## Interfaces produced (later epics consume these - restated here)

### `Repository` (`grepify.repository.base`)

Backend-neutral. v1 = JSONL truth + rebuilt SQLite cache; v2 = Postgres. **No
backend types in any signature.** Writes are idempotent by primary key; rebuild
is deterministic (same truth → same cache).

```python
class Repository(ABC):
    # truth writes (append-only, idempotent) - return count actually written
    def add_items(self, items: Sequence[Item]) -> int: ...
    def add_item_keywords(self, keywords: Sequence[ItemKeyword]) -> int: ...
    def add_digest(self, digest: Digest) -> None: ...
    def log_fetch(self, entry: FetchLogEntry) -> None: ...
    def log_llm(self, entry: LlmLogEntry) -> None: ...
    # truth reads (deterministic order)
    def iter_items(self) -> Iterator[Item]: ...
    def iter_item_keywords(self) -> Iterator[ItemKeyword]: ...
    def iter_digests(self) -> Iterator[Digest]: ...
    def existing_item_ids(self) -> set[str]: ...
    # config projection + cache lifecycle + queries
    def load_config(self, groups: Iterable[SourceGroup], sources: Iterable[Source]) -> None: ...
    def rebuild_cache(self) -> None: ...
    def count_items(self) -> int: ...
    def count_item_keywords(self) -> int: ...
    def close(self) -> None: ...
```

Data commits (JSONL truth) go through `repository.commit.commit_data(repo_dir,
paths, message, *, branch, ...)` - rebase-retry with a bounded attempt budget;
`[skip ci]` in the message avoids re-triggering cron (the Actions concurrency
group itself is GRP-06).

### `ConfigProvider` (`grepify.config.provider`)

Backend-neutral. v1 = filesystem-YAML; v2 = DB-backed. Accessors return domain
models / config schemas - **no filesystem types leak.**

```python
class ConfigProvider(ABC):
    def groups(self) -> list[SourceGroup]: ...      # domain models
    def sources(self) -> list[Source]: ...          # flattened; url + url_hash resolved
    def keywords(self) -> KeywordsConfig: ...        # aliases / mutes / pins
    def settings(self) -> SettingsConfig: ...        # llm profiles, windows, limits, tz
    def validate(self) -> ValidationReport: ...      # aggregates every problem, never raises
```

`validate` rejects duplicate source ids / url_hashes, bad kinds, missing
category (pydantic `extra="forbid"` catches typos). Live feed pinging is
deferred to when a fetcher exists (GRP-10/11).

### Domain models (`grepify.models`)

Pydantic records mirroring PRD §6 exactly: `Source`, `SourceGroup`, `Item`,
`ItemKeyword`, `KeywordAlias`, `Digest`, `FetchLogEntry`, `LlmLogEntry`, plus
`RunManifest`. Timestamps are ISO-8601 **strings** (§6 text columns; readable
JSONL diffs; Postgres-swappable).

## Tooling standards (enforced by `make check`)

- Python 3.12, `uv`. `ruff` (lint + format) and `mypy --strict` on `grepify/`;
  `pytest` for tests.
- SQL lowercase, no `select *`, explicit column lists.
- No SQLite/filesystem types in interface signatures (Postgres-swappable).
- Every module documents its **failure modes** in its docstring.
- `make check` = `lint` + `typecheck` + `test`; it is the per-issue DoD gate.
