# E3 - Site v1 (M3) - brief: build contract (DB+config → public/), page inventory, style tokens

> ~1 page, written once for the epic. Its job: let an agent implement any E3
> issue (and the E4 digest/keyword pages, which reuse the render machinery and
> trend queries) without reading other epics' code. PRD §5-§10, §12 are the
> source of truth; this restates the interfaces E3 produces and consumes.

## Goal

Turn the derived cache + config into a byte-stable static site under `public/`.
The whole epic hangs off two design-critical pieces built first - the **Jinja
SSG skeleton** (GRP-30: base layout, dark mobile-first, style tokens,
inline-SVG sparkline helper) and the **trend queries module** (GRP-31: cloud
dataset with deltas, stats, top sources, latest lists - pure window-
parameterized SQL over the cache). The pages (home GRP-32, items browser
GRP-33, sources + health GRP-34), the build command + Pages deploy (GRP-35),
and the GitLab CI mirror (GRP-36) plug into these. The site is a **pure
function of the SQLite cache + config** (F-SIT-08): same inputs → byte-stable
output, so snapshot tests are the regression net.

## Build contract (DB + config → public/)

```
sources/*.yml ─┐
               ├─▶ Repository.rebuild_cache()  (JSONL truth → grepify.db)
data/ (truth) ─┘            │
                            ▼
                   TrendQueries(conn, rules)  +  ConfigProvider  +  health.json
                            │
                            ▼
                   render (Jinja env, injected Clock + run_id)
                            │
                            ▼
                        public/           (index.html, items/, sources/, health/, static/)
```

- **Inputs**: the rebuilt SQLite cache (`data/grepify.db`), the `ConfigProvider`
  (sources/groups/keywords), and `data/health.json` (GRP-16). Nothing else.
- **Emission window (F-SIT-03/F-SIT-04, §9)**: pages emit only the **trailing
  90 days** of items; older data stays queryable via the cache/DB but is not
  paginated into `public/` (bounds build time + output size at 50k+ items).
  The cloud/stats windows (`settings.windows.cloud_days`, default 7) are
  independent and narrower.
- **Output**: `public/` - `index.html` (home), `items/` (paginated + emitted
  JSON for client-side filters), `sources/`, `health/`, and `static/`
  (`style.css`, `filters.js`). Byte-stable given the same inputs.

## Interfaces consumed (restated)

### The derived cache (v1 SSG reads it directly)

The trend queries read the **SQLite cache** (`DataLayout(data_root).cache_db`)
directly, *not* through `Repository`. This is deliberate and does not break
v2-proofing: PRD §15 replaces the entire SSG serving layer with FastAPI +
Postgres at v2, so the v1 SSG is intentionally SQLite-aware and thrown away at
the v2 boundary - the `Repository` interface stays backend-neutral for the
*pipeline*, which is what v2 actually preserves. The build orchestrator
(GRP-35) calls `Repository.rebuild_cache()` first, then opens a read connection
to the same cache file for rendering.

Cache schema is PRD §6 (`grepify.repository.schema.SCHEMA_DDL`): `items`,
`item_keywords`, `sources`, `source_groups`, `digests`, `fetch_log`, `llm_log`.
All timestamps are ISO-8601 UTC second-precision strings (`…+00:00`) - every
fetcher normalizes to `to_iso(...)`, so lexicographic string comparison **is**
chronological comparison; window bounds are computed with the same `to_iso`.

### `ConfigProvider` (`grepify.config`)

```python
def sources(self) -> list[Source]: ...       # rendered on the sources page (GRP-34)
def groups(self) -> list[SourceGroup]: ...    # category grouping
def keywords(self) -> KeywordsConfig: ...     # aliases/mutes → KeywordRules (trend merge)
def settings(self) -> SettingsConfig: ...     # .windows.cloud_days, tz
```

### Keyword rules (`grepify.keywords`) - GRP-23

```python
class KeywordRules:  alias_map: dict[str,str]; mute_set: frozenset[str]
    @classmethod
    def from_config(cls, cfg: KeywordsConfig) -> KeywordRules: ...
    def apply(self, raw: str) -> str | None: ...   # normalize→alias→mute; None if muted
```

Alias/mute is applied **at trend-computation time** (PRD §6), so remaps are
retroactive and non-destructive: the cloud/stats fold every windowed keyword
row through `rules.apply()` before counting.

### `Clock` (`grepify.clock`) + run id (`grepify.run`)

`Clock.now()` is the **only** wall-clock source in the render path (PRD §5 / S8
determinism). Templates never call `datetime.now()`; `generated_at` and the
cloud window are derived from the injected clock. `to_iso(instant)` is the
stable serializer.

### Health snapshot (`grepify.health`) - GRP-16

`HealthSnapshot` (`run_id`, `generated_at`, `sources: list[SourceHealth]`) read
from `data/health.json` and rendered on the health page (GRP-34).

## Interfaces produced (GRP-32..36 / E4 consume these)

### Render skeleton (`grepify.site`) - GRP-30

```python
STYLE_TOKENS: dict[str, str]                 # design tokens → CSS custom properties
def sparkline_svg(values, *, width=..., height=..., ...) -> str    # inline SVG, deterministic
def create_environment() -> jinja2.Environment                     # autoescape, deterministic
@dataclass(frozen=True)
class SiteMeta:  title: str; base_path: str; generated_at: str; run_id: str
@dataclass(frozen=True)
class PageContext:  meta: SiteMeta; nav: tuple[NavLink, ...]; ...
def render_page(env, template, ctx, **data) -> str                 # bytes-stable html
```

- **Style tokens** are the single source of truth for the palette/spacing/type
  scale (`tokens.py`); `style.css` is rendered from them into `:root{…}` CSS
  custom properties. Dark, mobile-first, **no external fonts, no trackers**
  (F-SIT-07) - system font stack only.
- **Sparkline** is a pure function (list of numbers → `<svg>` string) with
  fixed-precision coordinate rounding so output is byte-stable; used by keyword
  pages (E4/GRP-44), shipped now (GRP-30 scope).

### Trend queries (`grepify.site.trends`) - GRP-31

```python
@dataclass(frozen=True)
class Window:  start: str; end: str; days: int
def window_ending_at(instant, *, days) -> Window
def previous_window(window) -> Window

class TrendQueries:
    def __init__(self, conn: sqlite3.Connection, rules: KeywordRules) -> None: ...
    def cloud(self, window, *, previous=None, limit=...) -> CloudDataset       # counts + deltas
    def stats(self, window) -> Stats                                           # F-SIT-01 block
    def top_sources(self, window, *, limit=...) -> list[SourceCount]
    def latest_items(self, *, limit=..., since=None) -> list[ItemSummary]
    def latest_digests(self, *, limit=...) -> list[DigestSummary]
```

- **Mention count** for a keyword = **distinct items** mentioning it in the
  window (so an `llm` row and a `fallback` row for the same keyword on the same
  item count once, per §6's method-in-PK design).
- **Cloud** is sorted by count desc, then keyword asc (stable); `delta` = count
  − previous-window count. Rising detection (F-TRD-03) is **E4/GRP-40**, not
  here.
- **Stats**: `item_count`, `source_count`, `keyword_count`, `mention_count`
  (all in-window, after merge/mute), plus `top_keyword` / `top_source`.
- All SQL is lowercase, explicit-column, window-parameterized, with a total
  order (every `order by` carries a tie-breaker) so results are deterministic.

## Contracts & rules

- **Determinism (F-SIT-08, S8)**: no `datetime.now()` in the render path - the
  clock is injected; **all dict/set iteration is sorted** before it reaches a
  template; snapshot tests render **twice in a row** and assert byte-identical
  output. Coordinate/number formatting uses fixed precision.
- **Emission window**: only trailing-90d items are written to `public/`
  (F-SIT-03/04); the rule lives in the build orchestrator (GRP-35) and is
  passed to the item/keyword page emitters as a `since` bound.
- **Failure modes**: the build is offline and pure - it raises only
  `RepositoryError` (bad cache), `ConfigError` (bad config), or an `OSError`
  writing `public/`. A missing `health.json` renders an empty health page
  (health is best-effort), never fails the build. The LLM is never touched.
