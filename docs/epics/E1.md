# E1 - Ingestion core (M1) - brief: fetcher interface, item contract, dedup rules

> ~1 page, written once for the epic. Its job: let an agent implement any E1
> issue (and the E5 X/transcript fetchers) without reading other epics' code.
> PRD §5-§9, §12 are the source of truth; this restates the interfaces E1
> produces and consumes.

## Goal

Fill data into the store on a cron: fetch every enabled source, normalize each
raw entry into the PRD §6 `Item` shape, dedup it, and append it to JSONL truth.
The whole epic hangs off two design-critical contracts built first - the
**Fetcher** interface (GRP-10) that every source kind implements, and the
**normalizer + dedup** rules (GRP-14) that turn heterogeneous raw entries into
one stable, idempotent item identity. Concrete fetchers (RSS/YouTube/Reddit -
GRP-11/12/13), the orchestrator (GRP-15), and health (GRP-16) plug into these.

```python

```

### Domain models (`grepify.models`)

`Source` (source_id, name, kind, url, url_hash, group_id, enabled, added_at,
config_json) and `Item` (PRD §6 column set: item_id, source_id, kind,
external_id, canonical_url, title, summary, author, published_at, fetched_at,
content_hash, transcript_ref, lang). Timestamps are ISO-8601 **strings**.
`Clock` (`grepify.clock`) supplies `fetched_at`.

## Interfaces produced (later E1/E5 issues consume these)

### `RawItem` + `Fetcher` (`grepify.ingest`) - GRP-10

`RawItem` is a fetcher's output for **one** feed entry, *before* identity/hashing.
A fetcher does the kind-specific parsing (feed XML, channel RSS, reddit JSON,
tweet object); it never computes `item_id`/`content_hash`/`canonical_url` - that
is the normalizer's job, so identity rules live in exactly one place.

```python
class RawItem(BaseModel):          # frozen, extra=forbid
    external_id: str | None = None # guid / video_id / reddit id / tweet id (None if the feed gave none)
    url: str                       # raw link as provided by the source
    title: str
    summary: str | None = None     # raw description/selftext (normalizer truncates to 2k)
    author: str | None = None
    published_at: str | None = None# ISO-8601 if the source provided a date; None -> normalizer falls back to fetched_at
    lang: str | None = None
    transcript_ref: str | None = None  # youtube only; typically None until E5

class Fetcher(ABC):
    @property
    def kind(self) -> SourceKind: ...
    def fetch(self, source: Source) -> list[RawItem]: ...
```

`Fetcher.fetch` contract (the failure-isolation design GRP-15/16 depends on):

- Returns the source's current entries as `RawItem`s. An **empty** feed is a
  normal `return []`, not an error.
- Any **per-source** failure (timeout, HTTP error, malformed feed, auth
  challenge, rate limit) MUST be raised as `FetchError` (`grepify.errors`). It
  is non-fatal: the orchestrator (GRP-15) catches it, writes an `error`
  `fetch_log` row, and continues the run. One dead feed never fails the run
  (PRD §9). Unexpected exceptions are also caught by the orchestrator, but
  `FetchError` is the declared contract.
- Per-run per-source item caps (F-ING-06, default 50) are applied by the
  fetcher/orchestrator, not baked into this contract.

`FetcherRegistry` maps one fetcher per `SourceKind` and dispatches by
`source.kind`:

```python
reg = FetcherRegistry()
reg.register(RssFetcher())                 # keyed by fetcher.kind; duplicate kind -> ValueError
items = reg.fetch(source)                  # dispatch by source.kind; unknown kind -> KeyError (systemic, not FetchError)
```

`FakeFetcher` (shipped in `grepify.ingest`, not tests/) is the fixture-based
fake every E1/E2/E5 test uses: per-source canned `RawItem`s or a canned
`FetchError`, and a recorded `.calls` list.


## Contracts & rules

- **Identity (item_id).** `item_id = sha256(kind + "\0" + identity)` where
  `identity = external_id` when the entry carries one, else the `canonical_url`
  (PRD §6 `kind + canonical_url|external_id`). Deterministic and independent of
  `fetched_at`, so re-fetching the same entry yields the same `item_id` →
  `add_items` writes zero new rows. This is what makes ingest idempotent
  (F-ING-07); it is the AC test for GRP-14.
- **Unique-index handling.** The cache carries `unique index idx_items_dedup on
  items(kind, external_id)`. When `external_id` is present it *is* the identity,
  so two entries sharing `(kind, external_id)` collapse to one `item_id` before
  insert - the unique index can never be violated by construction. Entries with
  no guid store `external_id = NULL` (SQLite treats NULLs as distinct, so they
  don't collide) and dedup on `canonical_url` via `item_id`. The normalizer
  coerces an empty/whitespace `external_id` to `None` so blank ids can't
  masquerade as a shared non-null key.
- **canonical_url.** Conservative, deterministic canonicalization: lowercase
  scheme+host, drop default port, drop fragment, strip tracking query params
  (`utm_*`, `gclid`, `fbclid`, `ref`, …) preserving the order of the rest, strip
  a single trailing slash. Non-`http(s)`/relative URLs pass through unchanged.
- **content_hash (near-dups).** 64-bit **simhash** over the normalized title
  (lowercased, entities unescaped, tags/punctuation stripped, whitespace
  collapsed, unigram tokens), hashed with `blake2b` (process-independent - never
  Python's salted `hash()`), stored as 16-char hex. Near-dups = small Hamming
  distance. `group_near_duplicates` clusters a **windowed/paginated** set (it is
  O(n²); the site collapses per page/window, not the whole corpus - PRD §6
  note 2, F-SIT-03) and never deletes - grouping only.
- **Field hygiene.** `summary` truncated to 2000 chars (§6, F-ING-04). Fetchers
  own display-ready `title` text and ISO-or-`None` dates; the normalizer only
  normalizes internally for hashing.
- **Failure modes.** Per-source fetch failure → `FetchError` (isolated, logged,
  run continues). Missing/duplicate registry registration → `KeyError`/
  `ValueError` (systemic programming errors, loud). Normalizer is pure and
  raises only `pydantic.ValidationError` if a `RawItem` yields a field the
  `Item` model rejects.
